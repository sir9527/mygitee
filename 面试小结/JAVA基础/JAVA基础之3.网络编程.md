## 2.TCP的3次握手和4次挥手

```
1.  -->  SYN 请求B打开端口号
2. <--   SYN请求A打开端口号、ACK确认
3.  -->  ACK 确认

1. --> FIN   请求关闭B端口号   （A不能再给B发送信息）
2. <-- ACK   确认（确认消息发送后，可能B还会给A发送消息）
3. <-- FIN   请求关闭A端口号   （B不能再给A发送信息）
4. --> ACK   确认
```
